What Are React Hooks?

React Hooks are superpowers for functional components!

Traditionally, functional components (like the one at the bottom of this slide) have been relegated to dumb views.

This means they didn't have access to any local, changeable, state or any lifecycle methods (e.g., componentDidMount).

Functional components could only render the data that was passed in via props.

And they look something like this:

         
            const PirateShip = props => (
              <div>
                <h1>I'm a Pirate Ship</h1>
                <p>I have {props.masts} masts.</p>
                <p>I have {props.cannons} cannons.</p>
                <p>And, I have {props.crewmembers} able-bodied crewmembers.</p>
              </div>
            )

Hooks Change That ðŸ˜Ž

Hooks are functional way to work with many different changes in functional components.

In their simplest form, they give functional components access to local state and something akin to lifecycle methods, which makes functional components capable of so much more!

Here we change that PirateShip from the last slide to use hooks:

        
            import React, { useState } from 'react'

            const PirateShip = _ => {
              const [pirateShip, setPirateShip] = useState({
                masts: 2, cannons: 12, crewMembers: 30
              })
              
              return (
                <div>
                  <h1>I'm a Pirate Ship</h1>
                  <p>I have {pirateShip.masts} masts.</p>
                  <p>I have {pirateShip.cannons} cannons.</p>
                  <p>And, I have {pirateShip.crewMembers} able-bodied crewmembers.</p>
                </div>
              )
            }

            export default PirateShip

This example shows off the `useState` hook.

It takes the initial state as argument and returns the current state object and a function to change that state.

What's wrong with my class-based components?

  • Overhead
  • More code that's less shareable 
                  
                  import React from 'react'
      
                  class PirateShip extends React.Component {
                    constructor(props) {
                      super(props)
                      this.state = {
                        masts: 0,
                        cannons: 0,
                        crewMembers: 0,
                      }

                      this.getMyShip = this.getMyShip.bind(this)
                    }

                    componentDidMount() {
                      this.getMyShip()
                    }

                    componentDidUpdate() {
                      this.getMyShip()
                    }

                    getMyShip() {
                      // Call an api to get ship details
                    }

                    render() {
                      // ...
                    }
                  }

                  export default PirateShip
  • `this`
  • Classes in ES6 just aren't great as a feature of the language: References
  • Have to remember to use a constructor, super, and bind all of your methods.
  • These are fine in components you've already written, so don't throw away everything and rewrite in hooks. But, maybe, consider writing new components using hooks and functional components.
