React Hooks

A Little Background

  • Before we launch into hooks, I'll give a little background on why we might we might want hooks at all.
  • The main impetus for hooks is class-based components vs functional components.

Functional Components

  • Simple, JavaScript-y way to define components.
  • It's just a function that returns a chunk of JSX.

Example:

          
            const PirateShip = props => (
              <div>
                <h1>I'm a Pirate Ship</h1>
                <p>I have {props.masts} masts.</p>
                <p>I have {props.cannons} cannons.</p>
                <p>And, I have {props.crewmembers} able-bodied crewmembers.</p>
              </div>
            )

Class-based Components

  • Use ES2015 classes that inherit from React to create components.
  • They allow a component to manage its own state, as well as define how it behaves at different points in its life.
  • However, classes in JavaScript are a little deceptive. It's outside the scope of this talk to go into why, so I'll leave this link here with an excellent explanation and list of resources that do a fantastic job of explaining why I would prefer to avoid ES2015 classes.

Example:

        
          class PirateShip extends React.Component {
            constructor(props) {
              super(props)
              this.state = {
                masts: 2,
                cannons: 20,
                crewmembers: 50,
              }
            }

            render() {
              return (
                <div>
                  <h1>I'm a Pirate Ship</h1>
                  <p>I have {this.state.masts} masts.</p>
                  <p>I have {this.state.cannons} cannons.</p>
                  <p>And, I have {this.state.crewmembers} able-bodied crewmembers.</p>
                </div>
              )
            }
          }

So... Why would I ever use Class-based Components

  • Well, really, because React requires it.
  • Class-based components are the only way to have local component state.
  • Class-based components are the only to have access to the lifecycle methods.
  • Functional components were relegated to use for "dumb view" components that could only use the data they were given.
  • Until today!

React Hooks

  • React hooks give functional components superpowers!
  • Hooks make it possible to use local component state and something akin to lifecycle methods in a functional component!
  • Here's how it works:
        
          import React, { useState } from 'react'

          const PirateShip = () => {
            const [state, setState] = useState({masts: 2, cannons: 20, crewmembers: 50})

            return (
              <div>
                <h1>I'm a Pirate Ship</h1>
                <p>I have {state.masts} masts.</p>
                <p>I have {state.cannons} cannons.</p>
                <p>And, I have {state.crewmembers} able-bodied crewmembers.</p>
              </div>
            )
          }
  • This is the same component as the class-based version above.
  • useState just takes the initial state for the component and returns the state and the function used to set new state.
  • And, this is much simpler code. I don't have to remember the constructor or the separate render function inside the class. I don't have to deal with the weirdness that is ES6 classes.
  • This enables functions to much more viable throughout an application!
